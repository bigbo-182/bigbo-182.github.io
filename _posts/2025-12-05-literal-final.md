# Java 리터럴과 상수 완전 정리 – 면접 대비 기초 개념

자바에서 변수를 이해하기 위해 반드시 알아야 하는 개념이 바로 **리터럴(literal)**과 **상수(constant)**입니다.  
두 개념은 비슷해 보이지만 역할·저장 방식·사용 목적이 명확히 다르며, 면접에서도 자주 등장하는 중요한 파트입니다.

---

# 🔍 1. 리터럴(Literal)이란?

**리터럴 = 코드에서 값 그 자체를 의미하는 표기 방식**

예: 정수 리터럴, 실수 리터럴, 문자열 리터럴, boolean 리터럴, 문자 리터럴 등이 존재한다.

## ✔ 리터럴의 특징

- 값 그 자체를 코드에 직접 표현한 형태  
- 변수에 대입되거나 연산에 직접 사용  
- 메모리의 "상수 풀(Constant Pool)" 또는 literal pool에서 관리될 수 있음  
  (특히 **String 리터럴은 String Pool**에 저장됨)

---

# 🔍 2. 리터럴의 종류

| 리터럴 종류 | 예시 |
|-------------|------|
| 정수 리터럴 | `10`, `0x3F`, `0b1010` |
| 실수 리터럴 | `3.14`, `1.5e10` |
| 문자 리터럴 | `'A'`, `'가'` |
| 문자열 리터럴 | `"Hello"` |
| boolean 리터럴 | `true`, `false` |
| null 리터럴 | `null` |

---

# 🔥 3. 상수(Constant)란?

**프로그램 실행 중 값이 절대 변하지 않는 데이터**

자바에서는 **`final` 키워드**를 사용해 상수를 선언한다.

예:
`final int MAX_USER = 10;`

## ✔ 상수의 특징

- 한 번 초기화되면 값 변경 불가  
- 반드시 초기화 필요 (선언 시 또는 생성자에서)  
- 일반적으로 **대문자 + 스네이크 케이스**로 작성  
  - 예: `MAX_COUNT`, `DEFAULT_SPEED`

---

# 🔥 4. 상수 vs 리터럴 차이 (중요)

| 구분 | 리터럴 | 상수(Constant) |
|------|--------|----------------|
| 의미 | 값 그 자체 | 변경 불가한 변수 |
| 형태 | `10`, `"A"` | `final int MAX = 10` |
| 저장 위치 | Constant Pool 등에 저장될 수 있음 | 변수로 선언되어 메모리에 저장 |
| 용도 | 단순 값 표현 | 재사용 / 의미 전달 / 코드 가독성 향상 |

📌 **핵심 요약**  
- **리터럴은 “값 그 자체”**  
- **상수는 “값을 변경할 수 없도록 만든 변수”**

---

# 🔍 5. Java에서 상수를 사용하는 이유

- 코드 의미를 명확하게 만들기 위해  
- 매직 넘버(magic number) 제거  
- 값 변경 시 한 곳에서만 수정 가능  
- 협업 시 의도 전달이 쉬움  

예:

`int time = 60 * 60 * 24;` → 의미 불명확  
`final int SECONDS_PER_DAY = 60 * 60 * 24;` → 명확하고 안전

---

# 📌 마무리

리터럴은 “표현된 값 자체”,  
상수는 “변경할 수 없도록 정의된 변수”.
