# Java 면접 대비 – JVM 메모리 구조 & 자료형 핵심 질문 정리

이 글은 **JVM 메모리 구조, 기본 타입, 참조 타입**에 대해 공부하면서 정리한 내용을 기반으로,  
해당 주제에서 실제 면접에서 자주 나오는 질문과 핵심 답변만 모아놓은 정리입니다.

계속 공부할 때마다 여기에 내용을 추가해 나갈 예정입니다.

---

# 🔥 1. JVM 메모리 구조 관련 질문

---

## ❓ Q1. JVM 메모리 구조를 설명해보세요.

### 💬 핵심 답변
JVM 메모리는 크게 **Method Area, Heap, Stack**으로 구성됩니다.

- **Method Area**: 클래스 정보, static 변수, static 메서드 저장  
- **Heap**: new로 생성된 객체가 저장되는 공간, GC 대상  
- **Stack**: 지역변수, 매개변수, 참조 주소값이 저장되는 공간(메서드 호출 시 생성)

힙은 가비지 컬렉션이 관리하고, 스택은 메서드 종료 시 자동으로 제거됩니다.

---

## ❓ Q2. Heap과 Stack의 차이를 설명해보세요.

### 💬 핵심 답변
- **Stack**  
  - 메서드 실행 시 생성되는 스택 프레임 저장  
  - 지역 변수, 매개변수, 기본 타입 값, 참조 타입의 주소값 저장  
  - 메서드 종료 시 자동 제거  

- **Heap**  
  - new로 생성된 객체가 저장되는 공간  
  - GC가 참조되지 않는 객체를 정리  

---

## ❓ Q3. Stack이 GC 대상이 아닌 이유는?

### 💬 핵심 답변
스택 프레임은 메서드가 종료되면 즉시 제거되므로  
GC가 필요하지 않습니다.  
GC는 오직 Heap 영역에서만 동작합니다.

---

## ❓ Q4. static 변수는 어디에 저장되나요?

### 💬 핵심 답변
static 변수는 **Method Area**에 저장됩니다.  
클래스 로딩 시 초기화되며 프로그램 종료까지 유지되고, 모든 인스턴스가 공유합니다.

# 🧱 2. 기본 타입(Primitive Type) 관련 질문

---

## ❓ Q5. 자바 기본 타입은 무엇이 있고 각각의 크기는 어떻게 되나요?

### 💬 핵심 답변
자바 기본 타입은 총 8가지입니다.

| 타입 | 크기 | 범위 |
|------|------|----------------------------|
| byte | 1 byte | -128 ~ 127 |
| short | 2 byte | -32,768 ~ 32,767 |
| int | 4 byte | 약 ±21억 |
| long | 8 byte | 매우 큼 |
| float | 4 byte | 약 ±3.4e−38 ~ ±3.4e38 |
| double | 8 byte | 약 ±1.7e−308 ~ ±1.7e308 |
| char | 2 byte | 유니코드 문자 저장 |
| boolean | JVM 구현 의존 | true / false |

특징: **기본 타입은 스택에 값 자체가 저장**됩니다.

---

## ❓ Q6. boolean의 크기가 “JVM 의존”이라는 건 무슨 뜻인가요?

### 💬 핵심 답변
자바 스펙에서는 boolean의 바이트 크기를 명시하지 않았기 때문입니다.  
배열에서는 1바이트로 처리되지만 JVM 내부 최적화에 따라 1~4바이트로 정렬될 수 있습니다.

---

# 🧩 3. 참조 타입(Reference Type) 관련 질문

---

## ❓ Q7. 참조 타입이란 무엇인가요?

### 💬 핵심 답변
스택에는 **주소값(reference)**이 저장되고,  
실제 객체는 **힙(Heap)**에 저장되는 타입을 말합니다.

종류:
- 클래스(Class)
- 배열(Array)
- 인터페이스(Interface)
- Enum
- String

---

## ❓ Q8. 기본 타입과 참조 타입의 가장 큰 차이는?

### 💬 핵심 답변
- **기본 타입** → 스택에 값 자체 저장 (빠름, 메모리 작음)  
- **참조 타입** → 스택에는 주소값, 힙에는 실제 객체 저장  

---

## ❓ Q9. String은 기본 타입인가요? 참조 타입인가요?

### 💬 핵심 답변
`String`은 **참조 타입**입니다.  
문자열 리터럴은 별도로 관리되는 **String Pool**에 저장되지만,  
new String()은 Heap에 객체로 생성됩니다.
